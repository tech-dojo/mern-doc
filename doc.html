Prerequisites

Before you begin, you should make sure you have installed all these prerequisites on your development machine.

Node.js & npm Install

  • $ sudo apt-get install curl
  • $ curl -sL https://deb.nodesource.com/setup | sudo bash -
  • $ sudo apt-get install -y nodejs
  • $ sudo apt-get install -y build-essential (Recommended for Linux)
  • $ npm install -g gulp
  • $ npm install --save gulp gulp-live-server

MongoDB Install

  • $ sudo apt-key adv --keyserver hkp://keyserver.ubuntu.com:80 --recv 7F0CEB10
  • $ echo "deb http://repo.mongodb.org/apt/ubuntu
  • $(lsb_release -sc)"/mongodb-org/3.0 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org-3.0.list
  • $ sudo apt-get update
  • $ sudo apt-get install -y mongodb-org

Git Install/Clone

  • $ sudo apt-get update
  • $ sudo apt-get install git
  • $ sudo git clone https://github.com/(Clone ReactFramework)

After cloning the project from github to run the app

    Make sure mongodb is running, to run mongodb

  • $ mongod


    • Install global dependencies with

  • $ npm install -g gulp bower react-tools browserify


    • Install dependencies with

  • $ npm install


    • Run app with

  • $ gulp serve


Architecture & Data Flow

The application architecture used on the client-side for data flow between the React components and the backend is a stripped down version inspired by Facebook’s Flux architecture, and has two main parts - stores and views.

Flow of Data

Let’s start with the Article list view component from the framework. In order to get the list of articles from the backend and load in the view, first ArticleStore.fetchArticleList() function is called when the component is initialized.

class ArticleList extends React.Component { constructor(props, context){ ... ArticleStore.fetchArticleList(); ... } ... } FIG: Article List Component

The fetchArticleList() in the ArticleStore.jsx makes an ajax GET call in order to get the data from the database using the appropriate API call.


  • So when this article list data returns successfully, how is this data rendered in the article list view?

    • _onChange() { this.setState(getArticleList()); } FIG: onChange function in view

    The _onChange(){...} function in the view is registered in the store as a callback function to be called when the data is available.

    componentWillMount() { ArticleStore.onChange(this._onChange); } FIG: onChange function being registered

    function onChange(listener) { changeListeners.push(listener); } FIG: onChange function in store

    Listeners being pushed to a changeListener array which is used below, The triggerListeners(){...} function is called when the data is successfully fetched so the registered changeListeners are run.

    function triggerListeners() { changeListeners.forEach(function(listener) { listener(); }) }; FIG: Running each registered listener function

    function fetchArticleList() { get("/api/articles").then((data) => { articleList = data; triggerListeners(); }); } FIG: fetchArticleList() calls the triggerListeners() when data is retrieved
